register
    1. клиет вводит свои данные
        - валидация фронт
        - отпрака
        - валидация бэк
    2. сервер генирирует пароль
        - записывает в таблицу (clientRegister)
        - отправляет сообщение на почте
    3. клиет проверяет почту, вписывает в инпут код
        - фронт отправляет данные на бэк, если верно клиент попадает в личный кабинет
